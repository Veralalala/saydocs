# 協議修訂與生效規則

### 修訂

平台可能因**監管、風控、精算模型或產品迭代**更新保險條款、費率參數或技術規則。修訂內容通常透過：

* **應用內公示**（版本號、生效時間、摘要或全文）；
* **文件站點**；

等形式發布。

***

### 生效與「已承保保單」

| 情形                 | 一般原则                                                                                                                                                                                                        |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **已生效的 active 保單** | 通常以**承保時入庫的條款版本與參數快照**為準（如 `payout_amount`、到期時間、觸發閾值等），不因事後條款微調而單方面改變**已鎖定**的經濟條款；具體以正式法律文本為準。                                                                                                              |
| **新購 / 續保**        | **續保**在實作上多為 **replace**：先結束舊單（含退費規則），再依**新單**定價與條款生效；見[保單生命週期](../../bao-xian/bao-dan-sheng-ming-zhou-qi.md)。                                                                                              |
| **強制停售或熔斷**        | 營運側可限制**新承保**（如 `insurance_enabled`、市場級熔斷）；**不當然免除**已成立義務的執行規則，但可能受**賠付開關、資金充足度**等影響，見[定價模型](../../bao-xian/ding-jia-mo-xing.md)與[保險不承保情形一覽](../feng-xian-pi-lu/bao-xian-bu-cheng-bao-qing-xing-yi-lan.md)。 |
