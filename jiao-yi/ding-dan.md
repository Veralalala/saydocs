# 訂單

### 你常用的訂單類型

| 類型          | 適合場景                                                    |
| ----------- | ------------------------------------------------------- |
| **市價**      | 想**立刻**按當時市場的**標記價體系**成交，接受平台規定的**微小滑點**。               |
| **限價**      | 你心中有**目標價**；**只有**價格對你有利時，才會成交。                         |
| **止盈 / 止損** | 和**開倉主單**一起下，或之後補掛。價格觸達時，系統會**自動幫你平倉**或部分平倉，**減少盯盤壓力**。 |

**說明**：在 `orders` 的展示狀態裡，止盈止損**不會**以「一直掛著的一條 open 條件單」形式占滿你的訂單列表。它們會在建倉成功後記在**倉位保護規則**裡，**觸發**時才生成**一條已成交的平倉記錄**。介面會更乾淨，邏輯上仍然完整反映你的意圖。

***

### 滑點：買貴、賣便宜一點點

為應對行情跳動與成交風險，系統會在**標記價**上加上 **基點（bps）量級的滑點：買入 時，執行路徑會略貴**；**賣出** 時，則會略**便宜**。這是業內常見的**對價側保護**。滑點**以實際配置與介面揭露為準**。

* **開多 / 開空**：按你的買 / 賣方向，向不利側偏一點。
* **平多**：相當於**賣出**平倉，按**賣**向滑點。
* **平空**：相當於**買入**平空，按**買**向滑點。

**這對你意味著什麼**：\
你在 App 裡看到的**成交價**，可能和第三方行情**差幾個基點**。在計算盈虧與手續費名義時，系統都用這套**可重現**的規則，不是隨機改價。
