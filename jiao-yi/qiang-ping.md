# 強平

**強平（Liquidation）** 是永續合約裡**最被動**的一種平倉。當**你這筆逐倉的權益**相對 **標記價（Mark）已經低於協議要求的維持保證金**時，系統會**按規則強制了結**該倉位，把虧損限制在**該倉保證金**與風險引擎能承受的範圍內，**避免**虧損無序擴大，進而**影響其他使用者**與**協議穩定**。

### **什麼時候發生**

在**逐倉**模式下，當**權益**，也就是大致上的**保證金 + 該倉未實現盈虧**，再加上強平鏈路中的**清算罰金**因素後，**嚴格低於維持保證金要求**時，就會進入**可強平**狀態。

* **用什麼價強平**：強平**全程按標記價**與當時解析出的**有效維持率**來判斷、成交與記帳。它和 [定價](ding-jia.md) 使用同源 Oracle，**不是介面**看到的**盤口瞬時最優價**。
* 接近**維持線**時，**AI 助手**會盡量給你**提醒**。
* 真正觸發強平時，系統會在**標記價**下，按**當時有效規則**一次性把倉位**平掉**。

### 強平**前**你還能做什麼

| 行動            | 作用                                    |
| ------------- | ------------------------------------- |
| **追加逐倉保證金**   | 把**權益**拉離維持線，並把清算價往**更安全**的一側推。       |
| **降槓桿**       | 在協議允許時，減少借入可減輕倉位對維持線的壓力。              |
| **止損 / 主動平倉** | **先於**強平退出，**可能**跳過本輪**自動強平產生的清算罰金**。 |

### 如何理解清算價

* 清算價是根據**當前保證金**、**持倉數量、開倉均價**與**最小維持保證金率**等因素計算出的結果。
* **部分平倉、結算資金費、調整槓桿**，以及**有效保證金變動**，都會讓清算價改變。
