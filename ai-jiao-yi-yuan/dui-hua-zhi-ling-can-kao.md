# 對話指令參考

SayFi 的核心互動方式是**自然語言**。你不需要先背熟所有專業術語，也可以像對交易助理說話一樣表達意圖；助手會把你的話整理成**可執行的操作或清楚的選項**，當然**下單前仍會請你確認**。

本節提供**參考句型與關鍵字**。這**不是**唯一合法語法，只是幫你更快上手。

***

### 使用原則

1. **把目標講清楚**：說出標的大致名稱，例如 BTC、黃金，以及你能接受的**保證金**或**名目部位**範圍。
2. **把風險邊界講清楚**：包含你想使用的**槓桿**區間，以及**止盈 / 停損**想法，例如價格條件或「回撤多少就平倉」。
3. **執行前一定確認**：只要涉及開倉、平倉、加減倉、調整槓桿等操作，**都以你在介面上的確認為準**；助手**不會**在你未確認時自行成交。
4. **先問再動作**：如果你還不確定，可以直接說「**先不要下單**」或「**只分析，不下單**」，限制這一輪對話的行為。

***

### 參考：開倉與倉位操作

| 你想做什麼     | 可以這樣說（示例）                                      |
| --------- | ---------------------------------------------- |
| 市價開多 / 開空 | 「用 **XX USDC** 保證金，**5 倍**槓桿，**做多** BTC」       |
| 限價掛單      | 「**BTC 跌到 9xxxx** 時**做多**，保證金 **XX**，槓桿 **x**」 |
| 加倉 / 減倉   | 「在目前 BTC 多單上**再加** XX USDC」「**平掉一半**目前倉位」      |
| 一次全平      | 「**全部平倉** / **關掉這個幣的倉位**」                      |
| 調整保證金     | 「幫這個倉位**再補** XX USDC 保證金」                      |
| 止盈停損      | 「獲利到 **X%** 就停利」「跌到 **Y** 就停損」                 |

標的名稱可以用中文或常見英文代碼；**完整交易對仍以平台即時支援為準**，你也可以直接問助手「現在支援哪些交易對」。

***

### 參考：查資訊與帳戶

| 你想查什麼   | 可以這樣說（示例）                        |
| ------- | -------------------------------- |
| 目前持倉與盈虧 | 「我**現在有哪些倉位**？」「**浮動盈虧**多少？」     |
| 帳戶與費用   | 「**查看我的帳戶** / **最近手續費、資金費**情況」   |
| 交易規則    | 「**最低**要下多少保證金？」「**資金費率**多久收一次？」 |

***

### 參考：保險相關問法

若產品已對你開放投保入口，你可以用自然語言**詢問機制與條件**，但**保費、是否可保、保障期間**一律以**投保確認頁**為準：

* **保本險**和**利增險**各是什麼？
* 這個 BTC 倉位可以加**保本險**嗎？**保障多久**？

***

### 不建議這樣說

* 期待「**全自動、完全不用管**」就能賺錢：這和產品設計不符；助手提供的是**參考與執行輔助**，**決策權與簽名權**都在你手上。
* 把 **私鑰、助記詞、完整助記** 直接丟進對話：任何正規產品都不應要求你這麼做；**請不要在聊天裡傳送**。

***

### 關於語音與多語言

* **語音**：若客戶端已開啟語音輸入，口述內容和打字等效，仍會走同一套確認流程。
* **中英混用**：一般可混用標的名稱與簡單參數；若解析上有歧義，助手會再向你確認。
