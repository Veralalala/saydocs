# 子帳戶與錢包

SayFi 採用 **多 AI Trader** 結構：每個 Trader 對應 **一個獨立子帳戶**。你在側邊欄或帳戶清單裡看到的**多錢包 / 多帳戶**體驗，背後其實是一套套**彼此隔離的保證金與倉位**——適合拿來區分不同策略與風險偏好，而資產仍然維持 **非託管、鏈上自主管理**。

***

### 什麼是子帳戶

* **一個 Trader = 一個子帳戶**：每個子帳戶都有**獨立的保證金池**與**逐倉**倉位；某一個子帳戶的虧損或強制平倉，**不會**自動拖累其他子帳戶。
* **資金隔離**：子帳戶 A 的 USDC 與倉位，不會和子帳戶 B **共用**；你可以把不同 Trader 視為不同的「資金桶」來規劃。
* **對話隔離**：每個 Trader 都維護**獨立的對話歷史**與助手上下文，避免多套策略在同一段對話裡混在一起。

目前的產品限制如下：

* 每位使用者最多建立 **10 個**子帳戶，也就是 **10 個 Trader**。
* 合約目前為 **USDC 本位永續**。

***

### 和錢包的關係

* **你仍然是資產的真正控制者**。
* **登入與簽名**：不管你使用 **MetaMask、WalletConnect、Rabby** 等錢包，或透過 **Particle** 搭配 **Google / X / GitHub / Apple** 社群帳號登入，**真正的成交與資金操作都依賴鏈上授權與簽名**；自然語言只是互動介面，不能繞過你的簽名權。
* **子帳戶 / Agent 代簽情境**：在需要更流暢互動時，產品可能會針對金鑰資料採用 **加密儲存、權限隔離、最小暴露面** 等做法，**實際仍以官方安全說明與介面揭露為準**。

在進行大額操作前，仍建議你在**錢包與交易介面**自行核對授權範圍與合約地址。

***

### 常見用法

| 情境         | 建議                                              |
| ---------- | ----------------------------------------------- |
| 想測試新策略     | 新建一個 Trader，用**小額資金**在獨立子帳戶裡驗證想法。               |
| 想分開長線與短線   | 建兩個子帳戶、兩套設定檔，讓資金與對話彼此隔離。                        |
| 想釐清到底是誰在下單 | 盡量在同一個 Trader 的對話裡確認指令；切換 Trader 前，先確認目前子帳戶與餘額。 |

***

### 安全提醒

* **沒有傳統人工客服**：不要相信任何自稱「SayFi 官方客服」的人向你索取**私鑰、助記詞、轉帳或付費解凍**。
* **自行核對鏈上資訊**：若你對持倉或資金有疑問，請以應用內與鏈上可查資料為準。
