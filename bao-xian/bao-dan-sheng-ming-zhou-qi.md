# 保單生命週期

| 狀態          | 含義                                             |
| ----------- | ---------------------------------------------- |
| `active`    | 保障進行中                                          |
| `refunding` | 已啟動退費流程，待退費入帳後收尾                               |
| `expired`   | 到期**未觸發**責任，自然結束                               |
| `triggered` | **保險責任已成立**，進入賠付流程                             |
| `cancelled` | 因退保、替換、倉位關閉、系統取消等非保險觸發賠償原因結束；經 `refunding` 後退費 |

***

### 使用者動作與語義

* **`buy`**：該倉位、該險種**尚無** active 保單時，進行**新承保**。
* **`renew`**：**已有** active 保單時，採用**先退保，再開新單**的原則；保障期從**新單**起算，不是在原保單上直接續期。
* **`cancel`**：對 active 保單**主動退保**，通常會走 `active` → `refunding` → `cancelled`，與退費主鏈路一致。

***

### 典型路徑

* **正常持有**\
  `active` → 到期未觸發 → `expired`。這屬於 `NO_TRIGGER` 類結算；到期通常**不退**保費，詳見 [退費規則](tui-fei-gui-ze.md)。
* **觸發賠付**\
  `active` → 滿足觸發條件 → `triggered`。此時會生成保險責任。
* **退保 / 替換 / 因倉位結束而取消**\
  `active` → `refunding` → `cancelled`。

***

### 保單生命週期與倉位生命週期的關係

* 倉位**全平**、**使用者 TP / SL**、**沒有保險觸發的強平**等情況，都會導致 active 保單**被動取消**，並進入退費流程。
* 當一張保單因 **REVIVAL** 或 **PROFIT\_DOUBLE** 觸發時，**同倉另一險種**的 active 保單也會伴生進入取消 / 退費路徑。
